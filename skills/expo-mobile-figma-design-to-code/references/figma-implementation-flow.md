# Figma Implementation Flow

## Required order

1. Get structured design context for the exact node
2. Get screenshot for the same node or variant
3. Inspect tokens, spacing, hierarchy, and assets
4. Map the design to project primitives and tokens
5. Implement in project conventions
6. Compare result against the screenshot

## Rule

Do not jump directly from a Figma link to production code without design context and screenshot. The screenshot verifies appearance. The design context verifies structure.
